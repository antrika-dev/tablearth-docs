# Excel Add-in Integration

A Microsoft Excel task-pane add-in that lets users ask natural-language questions against the active selection.

## Supported platforms

- Excel for Windows (Microsoft 365 channel, build 1808+)
- Excel for Mac (Microsoft 365 channel, build 16.0+)
- Excel on the Web (excel.office.com)

The add-in is a standard Office Task Pane Add-in (manifest XML, hosted as static HTTPS assets). It loads `office.js` from Microsoft's CDN at runtime — no host-specific binaries.

## Distribution

Two paths:

- **Per-user / dev** — *Insert → Add-ins → Upload My Add-in* and pick the manifest.
- **Tenant-wide** — push the manifest via *Microsoft 365 admin center → Integrated apps*.

## End-user setup

1. Open Excel → ribbon → **TableAI** group → **Open TableAI**. Task pane opens to the activation form.
2. Enter:
   - **Tenant URL** — e.g. `https://yourcompany.antrika.com`
   - **Activation Key** — issued by tenant admin
3. Once activated (validated against `/emp/1/api/tableai/apikey/validate`), the pane unlocks. Both values persist in `localStorage` per add-in origin.

## Using it

The pane has three tabs:

| Tab | What it does |
|---|---|
| **Table Preview** | Shows columns + first 100 rows of the selected range. Download-CSV button. |
| **Ask Queries** | Streaming chat against the selected table. Markdown text + inline charts. |
| **Dashboard** | KPI cards + chart widgets + paginated data table generated by the backend after upload. Hidden until the spec is ready. |

### How it reads the table

`Excel.run` prefers, in order:

1. A structured **Excel Table** (Ctrl+T / `ListObject`) intersecting the active selection — uses the table's whole range.
2. The active range. If only one cell is selected, expands to the **current region** (Ctrl+Shift+8).

The first row is treated as headers. Rows are joined to RFC-4180 CSV in-memory.

### Session keying

```
remoteSessionId = 'tai-xl-' + sha256(csv).slice(0, 32)
```

Same data → same backend session → resumes prior chat. Edit the data → new session, fresh conversation.

## Architecture

The add-in is a standard Office Task Pane Add-in:

- **Manifest XML** — describes the ribbon button and the task pane URL. Office hosts (Windows / Mac / Web) load the pane from your HTTPS host.
- **Task pane** — three tabs (Preview · Ask · Dashboard) plus the activation gate. Fetches the backend directly using the activation key.
- **Ribbon command** — opens the pane from any sheet.

## Backend endpoints used

Same API-key surface as the Chrome extension. Full reference: [api/endpoints.md](../api/endpoints.md).

## Branding

Replace icons with branded versions before publishing — Microsoft recommends square PNGs at 16, 32, 64, 80, and 128 pixels.

## Mapping vs the Chrome extension

The two surfaces share the same backend lifecycle. Where they differ:

| Concept | Chrome extension | Excel add-in |
|---|---|---|
| Activation | Options page on first install | In-pane activation gate (single-page UX) |
| Table picker | Click a ✨ badge on any HTML table | Ribbon **Open TableAI** + in-pane **Use selected range** |
| Network | Service worker relays to avoid CORS | Pane fetches the backend directly |
| Storage | `chrome.storage.local` | `localStorage` (per add-in origin) |

Chat, dashboard, and chart rendering behave identically against the same backend payloads.

## Troubleshooting

| Symptom | Likely cause |
|---|---|
| Pane loads but stuck on activation form | Tenant URL wrong, or `/apikey/validate` returns non-200. Check the network tab. |
| `Use selected range` does nothing | Less than 2 rows × 2 cols selected, or selection is on an empty sheet. |
| Charts don't render | Backend returned a `CHART` event with a spec the renderer doesn't support yet. Falls back to data table. |
| Dashboard tab empty | Spec not yet generated. Backend job runs after first upload — refresh tab in ~30s. |

More in [troubleshooting.md](../troubleshooting.md).
